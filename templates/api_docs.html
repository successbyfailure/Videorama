Utiliza estos endpoints para automatizar descargas, convertir audio en subtítulos o integrar el servicio en tus herramientas. Todas las respuestas devuelven errores en formato JSON cuando ocurre algún problema.

Descarga directa

El endpoint GET /api/download acepta dos parámetros:

url
Enlace completo del video/audio soportado por yt-dlp.
format
Uno de los formatos listados arriba. Por omisión descarga video.
curl -L "${BASE_URL:-http://localhost:8601}/api/download?url=https://youtu.be/&format=audio" \
  -o audio.mp3

Operaciones con ffmpeg

Usa el endpoint POST /api/ffmpeg/upload para subir un archivo local y recodificarlo con uno de los presets ffmpeg_*. Puedes elegir entre ffmpeg_audio, ffmpeg_audio_wav, ffmpeg_1440p, ffmpeg_1080p, ffmpeg_720p y ffmpeg_480p. 

curl -X POST "${BASE_URL:-http://localhost:8601}/api/ffmpeg/upload" \
  -F "media_format=ffmpeg_1080p" \
  -F "file=@/ruta/al/video.mov" \
  -o master-1080p.mp4

El servidor no almacena el archivo original, solo la conversión temporal que se devuelve inmediatamente, y aplica los parámetros de ffmpeg predefinidos para cada preset.

Inspeccionar sin descargar

GET /api/probe?url=... consulta yt-dlp en modo dry-run y devuelve los metadatos (título, duración, categorías y miniaturas). Es ideal para que otros servicios, como la biblioteca de Videorama, clasifiquen el contenido sin esperar a la descarga.

Transcribir un archivo local

Envía un archivo de audio o video con POST /api/transcribe/upload usando multipart/form-data.

curl -X POST "${BASE_URL:-http://localhost:8601}/api/transcribe/upload" \
  -F "media_format=transcripcion_srt" \
  -F "file=@/ruta/al/audio.mp4" \
  -o subtitulos.srt

Los formatos válidos son transcripcion, transcripcion_txt y transcripcion_srt.

Gestionar la caché

  • GET /api/cache: lista los elementos disponibles, tamaños y tiempos de expiración.
  • GET /api/cache/{cache_key}/download: recupera un elemento específico.
  • DELETE /api/cache/{cache_key}: elimina un archivo y su metadato asociado.

Estadísticas y salud

  • GET /api/health: devuelve {"status": "ok"} cuando el servicio responde.
  • GET /api/stats/usage: totales, aciertos de caché, número de recodificaciones ffmpeg_*, métricas de transcripción y los formatos más solicitados en los últimos 7 días.
