Descarga directa

El endpoint GET /api/download acepta dos parámetros:

url
Enlace completo del video/audio soportado por yt-dlp.
format
Uno de los formatos listados arriba. Por omisión descarga video.
curl -L "${BASE_URL:-http://localhost:8000}/api/download?url=https://youtu.be/&format=audio" \
  -o audio.mp3

Transcribir un archivo local

Envía un archivo de audio o video con POST /api/transcribe/upload usando multipart/form-data.

curl -X POST "${BASE_URL:-http://localhost:8000}/api/transcribe/upload" \
  -F "media_format=transcripcion_srt" \
  -F "file=@/ruta/al/audio.mp4" \
  -o subtitulos.srt

Los formatos válidos son transcripcion, transcripcion_txt y transcripcion_srt.

Gestionar la caché

  • GET /api/cache: lista los elementos disponibles, tamaños y tiempos de expiración.
  • GET /api/cache/{cache_key}/download: recupera un elemento específico.
  • DELETE /api/cache/{cache_key}: elimina un archivo y su metadato asociado.

Estadísticas y salud

  • GET /api/health: devuelve {"status": "ok"} cuando el servicio responde.
  • GET /api/stats/usage: totales y aciertos de caché de los últimos 7 días.
